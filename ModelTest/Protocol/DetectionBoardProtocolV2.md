# 检测装置电测板卡通信协议 V2.0

## 协议组成

台体通信协议包括控制协议和透传协议。

控制协议用于控制台体模块的电源系统、遥信状态设置、遥控状态读取、电压电流等参数检测。

透传协议用于透传模块和终端、模块与虚拟表之间的通信报文或抄表报文，例如 645、698、376.2、376.3 等。

## 通信参数

控制协议采用 RS485 通信：

- 波特率：115200 B/S
- 校验：无校验
- 停止位：1 位

透传协议通信端口参数根据被测模块不同而不同。

## 报文格式

| 字段 | 长度 | 说明 |
| --- | --- | --- |
| 起始字符 | 2 字节 | 固定 `55 44` |
| 数据长度 | 2 字节 | 小端格式，表示从“数据长度”到“校验和”的长度，不包含起始字符和结束字符 |
| 数据方向 | 1 字节 | 下行到 MCU 为 `00`，上行到 PC 为 `01` |
| 地址/通道 | 1 字节 | 印制板地址，有效范围 `01`~`FE`，`FF` 为广播地址 |
| 协议类型 | 1 字节 | bit0 表示控制/透传，bit1~bit7 表示设备类型 |
| 命令码 | 1 字节 | 控制协议表示命令，透传协议表示软件透传通道 |
| 数据项 | N 字节 | 控制协议为命令附加数据，透传协议为透传数据 |
| 校验和 | 1 字节 | 从“数据长度”到“校验和前一个字节”的累加和低字节 |
| 结束字符 | 2 字节 | 固定 `AA BB` |

## 协议类型

协议类型字段按 bit 解释：

- bit0：`0` 表示控制协议，`1` 表示透传协议
- bit1~bit7：设备类型

设备类型：

| 值 | 设备 |
| --- | --- |
| 1 | 电表 |
| 2 | 终端 |
| 3 | 源 |
| 4 | 仓储 |
| 其他 | 预留 |

协议类型组装公式：

```text
协议类型 = (设备类型 << 1) | 是否透传
```

例如终端控制协议：设备类型 `2`，控制协议 `0`，协议类型为 `04`。

例如终端透传协议：设备类型 `2`，透传协议 `1`，协议类型为 `05`。

## 长度说明

数据长度为 2 字节小端格式。

长度范围从“数据长度”字段开始，到“校验和”字段结束，不包含起始字符 `55 44`，也不包含结束字符 `AA BB`。

最小无数据项报文的数据长度为：

```text
数据长度2 + 数据方向1 + 地址1 + 协议类型1 + 命令码1 + 校验和1 = 7
```

## 校验和

校验和为各字节累加和的低字节。

校验范围：

```text
数据长度字段低字节
数据长度字段高字节
数据方向
地址
协议类型
命令码
数据项
```

不包含起始字符、校验和自身、结束字符。

## 应答报文

设置命令：

- 成功：返回码按原命令返回
- 失败：按错误码返回

读取状态命令：

- 成功：返回码按规定格式返回
- 失败：按错误码返回

## 代码封装

项目中协议封装类：

```text
ModelTest/Protocol/DetectionBoardProtocolV2.cs
```

该类负责：

- 控制协议帧拼装
- 透传协议帧拼装
- 完整帧解析
- 起始符、结束符、长度、校验和校验
- 表位十进制转 16 进制

`TerminalV2UserControl` 只负责界面控件和调用协议类。
